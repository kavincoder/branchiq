# Product Requirements Document (PRD)
## BranchIQ — AI-Powered Branch Banking Management System

**Version:** 1.0  
**Date:** 2026-04-19  
**Author:** Project Team  
**Status:** Draft

---

## 1. Overview

BranchIQ is an internal web application designed for single-branch banking operations. It enables branch staff to track all financial transactions, manage loan/deposit/investment portfolios, and receive AI-powered insights — all from a single, fast, secure dashboard.

---

## 2. Goals

- Replace manual spreadsheet-based tracking with a centralized system
- Give the branch manager real-time visibility into branch financial activity
- Detect unusual transactions automatically using AI
- Generate instant Excel reports for auditing and review
- Simple, fast, and secure — no unnecessary complexity

---

## 3. Non-Goals (Out of Scope)

- No customer-facing features
- No real money movement or payment gateway integration
- No multi-branch support
- No mobile app
- No SMS or email notifications (v1)
- No multi-currency support

---

## 4. Users & Roles

| Role | Access Level | Description |
|---|---|---|
| **Branch Manager** | Admin | Full access — view all data, manage staff accounts, view AI reports, export Excel |
| **Staff** | Standard | Can log transactions, view branch-wide transactions, view own activity |

## 5. Features

### 5.1 Authentication
- Username + password login
- Passwords stored as bcrypt hashes (never plain text)
- JWT-based session management (token expires after 8 hours)
- Branch Manager can create/deactivate staff accounts
- No self-registration — accounts created by manager only

### 5.2 Dashboard
- Summary cards: Total Deposits, Total Withdrawals, Active Loans, Total Investments
- Recent transactions feed (last 10)
- AI anomaly alert banner (if any suspicious transactions detected)
- Monthly spending trend chart (Recharts line graph)
- Portfolio breakdown chart (Recharts pie chart)

### 5.3 Transaction Management
Track the following transaction types (all in INR ₹):

| Type | Fields |
|---|---|
| Deposit | Account No, Amount, Date, Staff, Notes |
| Withdrawal | Account No, Amount, Date, Staff, Notes |
| Transfer | From Account, To Account, Amount, Date, Staff, Notes |
| Loan Repayment | Loan ID, Amount Paid, Date, Staff, Notes |

- Staff can add new transactions
- Staff can view all branch transactions (not edit/delete)
- Branch Manager can edit and delete transactions
- Filter by: date range, transaction type, account number
- Search by account number or transaction ID

### 5.4 Portfolio Management

#### Loans
- Loan ID, Borrower Name, Principal Amount, Interest Rate, Start Date, End Date, Status (Active/Closed/Defaulted)
- Track repayment history per loan
- Show outstanding balance automatically calculated

#### Deposits
- Account Number, Holder Name, Deposit Amount, Interest Rate, Type (Fixed/Savings), Start Date, Maturity Date, Status

#### Investments
- Investment ID, Type (Govt Bond / Mutual Fund / FD), Amount, Date, Expected Return %, Maturity Date, Status

### 5.5 AI Features

#### Anomaly Detection
- Runs on transaction data using Isolation Forest algorithm (scikit-learn)
- Flags transactions that are statistically unusual:
  - Unusually large amounts compared to account history
  - Transactions at odd times
  - Rapid repeated transactions on same account
- Flagged transactions shown with a red warning label
- Branch Manager gets an anomaly summary on dashboard

#### Spending Insights
- Monthly comparison: this month vs last month (deposits, withdrawals, loan repayments)
- Top transaction categories by volume
- Loan repayment performance: % of loans with on-time payments
- Trend detection: "Withdrawals increased 32% this month"
- All insights shown as text + chart on a dedicated Insights page

### 5.6 Excel Export
- Export transactions (filtered or all) to .xlsx
- Export loan portfolio to .xlsx
- Export deposit portfolio to .xlsx
- Export investment portfolio to .xlsx
- File named automatically: `BranchIQ_Transactions_2026-04-19.xlsx`
- Generated by openpyxl on the backend, downloaded via browser

### 5.7 User Management (Manager Only)
- Create staff account (name, username, password)
- Deactivate staff account
- View staff activity log (which staff added which transactions)

---

## 8. Non-Functional Requirements

- **Performance:** Pages must load under 2 seconds
- **Security:** All API endpoints protected by JWT; passwords never stored plain text
- **Accuracy:** Financial calculations must be precise (use Python `Decimal`, not float)
- **Availability:** Local network deployment (single branch, not cloud required in v1)
- **Browser Support:** Chrome, Edge (latest versions)

---

## 9. Data & Privacy

- All data stays within the branch's local server or internal network
- No data sent to external APIs except for AI processing (done locally)
- Staff cannot see other staff's login credentials
- Transaction logs are permanent — deletions logged, not truly erased (soft delete)

---

## 10. Success Metrics

- Branch Manager can view full transaction history in under 5 clicks
- AI detects at least 1 anomaly per test dataset of 500 transactions
- Excel export generates in under 3 seconds
- Zero login security incidents in first 3 months
